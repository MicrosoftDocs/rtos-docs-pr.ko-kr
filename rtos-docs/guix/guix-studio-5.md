# <a name="chapter-5-guix-studio-screen-designer"></a>챕터 5: GUIX Studio 화면 디자이너

애플리케이션 화면 디자인은 GUIX Studio의 주요 목적입니다. 화면 디자인은 챕터 3의 앞부분에서 설명한 모든 다양한 보기를 통해 수행됩니다. 그러나 GUIX Studio에서 화면 디자인의 주요 요소는 ***대상 보기*** 이며, 여기에는 모든 화면 요소가 포함된 대상 표시에서 표시되는 것과 똑같은 방식으로 시각적으로 표시됩니다. 이러한 화면 요소는 간단한 마우스 및 단추 작업을 통해 선택, 이동, 크기 조정 등을 수행할 수 있습니다. 또한 선택한 개체에 대한 맞춤 및 Z 순서 단추 작업을 사용할 수 있습니다. 다음 하위 섹션에서는 GUIX Studio 화면 디자인의 다양한 기능에 대해 설명합니다. 

## <a name="creatingconfiguring-projects"></a>프로젝트 만들기/구성

GUIX Studio에서 프로젝트를 만드는 것은 간단합니다. ***새 프로젝트** _ 단추 또는 _*_프로젝트, 새 프로젝트_*_ 메뉴 선택 항목을 선택하기만 하면 됩니다. 그러면 GUIX Studio에서 _ *_프로젝트 구성_** 대화 상자를 표시합니다. 이 대화 상자에는 기본 표시 설정 및 GUIX Studio에서 생성된 코드를 찾을 위치에 대한 경로 정보가 지정되어 있습니다.

새 프로젝트가 만들어지면 프로젝트 구성 대화 상자가 표시됩니다. 여기서 개발자는 대상에서 사용할 수 있는 하드웨어 표시 수와 각 표시에 대한 속성을 지정합니다. 속성에는 표시의 논리적 이름, x/y 해상도, 색 농도 및 서식, 기타 표시 속성이 포함됩니다. GUIX Studio는 동일한 프로젝트의 여러 표시를 지원합니다. 추가 표시가 필요한 경우 포함된 디바이스의 표시 수와 일치하도록 ***표시 수** _ 필드를 변경해야 합니다. 프로젝트의 최대 표시 수는 4개입니다. _ *_그림 21_**에서는 프로젝트 구성 대화 상자를 보여 줍니다.

프로젝트 및/또는 표시 설정 수정은 ***구성, 프로젝트/표시** _ 메뉴 옵션을 사용하거나 프로젝트 또는 표시를 선택하여 마우스 오른쪽 단추로 클릭하고 _*_구성, 프로젝트/표시_*_ 를 선택하여 수행됩니다. 두 경우 모두에서 프로젝트 설정 및/또는 표시를 쉽게 변경할 수 있도록 _ *_프로젝트 구성_** 대화 상자가 표시됩니다.

![프로젝트 구성 대화 상자의 스크린샷](./media/guix-studio/config_project.png)

**그림 21**

디렉터리 그룹에는 Studio에서 생성한 C 원본 및 헤더 파일에 대한 기본 출력 디렉터리를 지정할 수 있습니다. 이러한 디렉터리는 일반적으로 한 컴퓨터에서 다른 컴퓨터로 또는 한 파일 시스템에서 다른 파일 시스템으로 쉽게 이동할 수 있도록 프로젝트 위치를 기준으로 하여 저장됩니다.

추가 헤더 필드는 사용자 지정 헤더 파일을 지정할 수 있는 위치입니다. 둘 이상의 헤더 파일이 필요한 경우 세미콜론을 사용하여 목록을 구분합니다.

Studio "애플리케이션 생성" 또는 "리소스 생성" 명령을 호출하면 이러한 원본 파일이 작성되는 기본 디렉터리가 됩니다. 물론 이러한 디렉터리 위치는 언제든지 출력 디렉터리 대화 상자에서 새 위치를 입력하여 재정의할 수 있습니다.

## <a name="using-properties"></a>속성 사용

앞에서 설명한 대로 선택한 위젯의 속성은 ***속성 보기** _에 표시됩니다. 모든 위젯에는 일반적인 속성 세트뿐만 아니라 특정 위젯 형식과 관련된 일부 속성도 있습니다. 예를 들어 단추 위젯에는 _ *_Pushed_** 속성이 있지만 창 위젯에는 없습니다. 위젯 속성의 일반적인 세트는 다음과 같습니다.

| ---------------- | ------------------------------------------------------------------------------------- |
| 위젯 형식    | 위젯의 형식이며, 참조용입니다.                                                                               |
| 위젯 이름      | 위젯의 이름이며, 위젯 만들기 함수에 전달되고 생성된 원본 파일의 변수 명명에 사용됩니다.               |
| 위젯 ID        | 위젯의 ID입니다. 이 ID 값은 자식 위젯에서 부모 화면으로 신호를 생성하는 데 사용됩니다.                            |
| 왼쪽             | 위젯의 맨 왼쪽 좌표                                                                                                 |
| 맨 위로              | 위젯의 맨 위쪽 좌표                                                                                                  |
| 너비            | 위젯의 너비(픽셀)                                                                                                      |
| 높이           | 위젯의 높이(픽셀)                                                                                                     |
| 테두리           | 위젯 테두리의 유형                                                                                                          |
| 투명      | 위젯이 부분적으로 투명한지 여부를 확인해야 합니다.                                                                       |
| 선택한 그리기    | 위젯이 처음에 선택한 상태로 그려져야 하는지 확인해야 합니다.                                            |
| 사용           | 최종 사용자가 위젯을 선택하거나 클릭할 수 있는지 여부를 확인해야 합니다.                                                    |
| 포커스 허용    | 위젯에서 포커스를 허용하는지 여부를 확인해야 합니다.                                                                                 |

| 선택한 채우기    | 선택한 채우기 색 리소스 ID                                                                                                |
| 그리기 함수    | 사용자 정의 사용자 지정 그리기 함수 이름입니다. 이 필드가 비어 있으면 해당 위젯 형식에 대한 표준 그리기 함수가 사용됩니다. |

텍스트 위젯은 텍스트 맞춤 스타일 및 글꼴 사양과 함께 문자열 ID 필드를 지원합니다. 각 위젯 형식과 사용 가능한 스타일에 대한 설명을 읽고 해당 위젯 형식에 대한 함수 매개 변수를 만드는 경우 추가 위젯 속성이 일반적으로 매우 직관적입니다.

## <a name="manipulating-widgets"></a>위젯 조작

위젯을 조작하려면 먼저 해당 위젯을 선택해야 합니다. 이는 ***대상 보기** _에서 위젯을 직접 클릭하거나 _ *_프로젝트 보기_** 위젯 트리에서 위젯을 선택하여 수행됩니다. 위젯을 선택하면 해당 위젯에 파선 윤곽선이 있습니다. 이 상태에서는 위젯을 클릭하여 부모 위젯의 원하는 위치로 끌어서 놓기만 하면 해당 위젯을 이동할 수 있습니다. 위젯이 최상위 위젯인 경우 위젯을 끌면 대상 표시에서 위젯의 초기 위치가 효과적으로 설정됩니다. 또한 언제든지 GUIX API를 사용하여 위젯을 이동하거나 크기를 조정할 수 있습니다.

위젯의 높이를 조정하려면 마우스를 위젯의 위쪽 가장자리에 놓고 마우스 포인터가 위/아래 화살표로 변경될 때까지 기다립니다. 이 시점에서 마우스 오른쪽 단추를 누른 채 마우스를 움직이기만 하면 위젯 높이를 변경할 수 있습니다. 위젯의 너비는 마우스 포인터를 위젯의 왼쪽 가장자리에 배치하여 비슷한 방식으로 조정할 수 있습니다. ***그림 24** _에서는 크기가 조정되고 부모 창의 왼쪽/위쪽 영역으로 이동된 "_*_단추_**" 위젯을 보여 줍니다.

## <a name="manipulating-multiple-widgets"></a>여러 위젯 조작

여러 위젯은 대상 보기에서 ***Ctrl*** 키를 누른 채 여러 위젯을 클릭하여 선택할 수 있습니다. 이렇게 하면 선택한 각 위젯의 주위에 파선 윤곽선으로 표시됩니다. 여러 위젯을 선택하는 경우 선택 그룹의 각 위젯은 동일한 부모 위젯의 자식이어야 합니다.

여러 위젯이 선택되면 선택한 위젯의 내부를 클릭하고 마우스 오른쪽 단추를 누른 채 마우스를 이동하여 동시에 이동할 수 있습니다. 또한 ***도구 모음** _의 맞춤 단추를 사용하여 선택한 위젯 그룹을 맞출 수 있습니다. _*_그림 25_*_ 에서는 선택한 "_*_단추_*_"와 "_*_새 단추_*_" 위젯을 모두 보여 주고, _*_그림 26_*_ 에서는 이러한 위젯을 선택하는 동안 _ *_왼쪽 맞춤_* * 단추 선택의 결과를 보여 줍니다.

## <a name="cutcopypaste-operations"></a>잘라내기/복사/붙여넣기 작업

***대상 보기** _에서 선택한 위젯을 표준 방식으로 잘라내고, 복사하고, 붙여넣을 수 있습니다. 위젯 및 화면을 한 프로젝트 내에서 복사하거나, 한 프로젝트에서 복사하여 다른 프로젝트에 붙여넣을 수 있습니다. _*_도구 모음_*_ 에는 잘라내기, 복사 및 붙여넣기 단추가 있습니다. [편집] 메뉴 옵션에도 동일한 옵션이 있습니다. 위젯을 붙여넣는 경우 새 위젯을 붙여넣기 전에 부모 위젯을 선택해야 합니다. _*_그림 27_*_ 에서는 "_*_단추_**" 위젯을 선택하고, 복사하고, 이 복사본을 동일한 창에 붙여넣은 결과를 보여 줍니다.

![잘라내기/복사/붙여넣기 작업의 스크린샷](./media/guix-studio/copy_paste_button.png)

**그림 27**

하나의 프로젝트 내에서 작업하는 경우 복사된 위젯에 필요할 수 있는 리소스가 항상 있으므로 한 프로젝트 내의 복사/붙여넣기는 일반적으로 간단합니다. 그러나 프로젝트 A에서 위젯을 복사하여 해당 위젯을 프로젝트 B에 붙여넣으면 리소스 종속성과 관련된 몇 가지 문제가 발생할 수 있습니다.

Studio 내에서 위젯을 복사하는 경우 Studio 애플리케이션에서 복사된 위젯에 필요한 리소스의 목록을 만들고, 실제 복사된 위젯 정보와 함께 Windows 클립보드에 복사되는 XML 형식의 이식 가능한 리소스 종속성 테이블을 생성합니다. 위젯을 다른 프로젝트에 붙여넣는 경우 Studio는 먼저 리소스 종속성 목록을 검사하고, 필요한 리소스가 아직 없는 경우 해당 리소스를 열려 있는 프로젝트에 추가합니다. Studio는 리소스 ID 이름을 기준으로 일치하는 리소스를 식별하고, 문자열 리소스의 경우 문자열 내용도 비교합니다. 일치하는 리소스가 있으면 Studio에서 붙여넣은 위젯의 리소스 ID를 업데이트하여 새 프로젝트의 리소스를 적절하게 사용합니다. 리소스가 없으면 해당 리소스가 추가됩니다.

Studio에서 위젯 붙여넣기 작업의 일환으로 리소스를 프로젝트에 추가할 때 Studio는 실제로 글꼴 및 픽셀 맵 리소스의 경우 리소스에 대한 링크를 추가합니다. 이 링크는 원본 프로젝트에서 생성되며, 붙여넣는 프로젝트의 프로젝트 위치를 기준으로 해당 리소스를 찾을 수 없는 경우 경고 메시지가 표시됩니다. 리소스 링크는 어쨌든 프로젝트에 추가되지만, 리소스 로드 오류를 제거하려면 글꼴 및 이미지 파일을 새 프로젝트 트리 아래의 적절한 위치에 수동으로 복사해야 할 수도 있습니다. Studio는 .ttf, .png 또는 .jpg 파일을 한 위치에서 다른 위치로 복사하지 않습니다.

이와 관련된 문제를 쉽게 방지할 수 있는 방법은 공유하려는 프로젝트 간에 일관된 디렉터리 구조를 유지하는 것입니다. 프로젝트 A에서 프로젝트 B로 쉽게 이동하려면 두 프로젝트에서 사용하는 그래픽 이미지와 글꼴을 각 프로젝트 폴더의 일관된 하위 디렉터리에 유지합니다.

## <a name="changing-z-order"></a>Z 순서 변경

## <a name="assigning-colors-fonts-and-pixelmaps"></a>색, 글꼴 및 픽셀 맵 할당

선택한 위젯의 속성 보기에서 색, 글꼴 및 픽셀 맵을 선택하는 것 외에도 리소스를 위젯에 할당하는 간단한 끌어서 놓기 방법도 지원됩니다. 이 기능을 사용하려면 리소스 보기에서 글꼴 색과 같은 리소스를 마우스 왼쪽 단추로 클릭하고 대상 보기에서 원하는 위젯 위로 끌기만 하면 됩니다. 마우스 왼쪽 단추를 위젯 위에 놓으면 리소스를 삭제합니다.

끌어서 놓기 방법을 사용하는 경우 색 리소스는 항상 위젯의 일반 배경색으로 할당됩니다. 선택한 색 또는 선택한 텍스트 색과 같은 다른 색은 속성 보기를 사용하여 할당해야 합니다.

마찬가지로 픽셀 맵 리소스는 픽셀 맵 표시를 지원하는 위젯의 "표준" 또는 "채우기" 픽셀 맵 필드에 할당됩니다. 여러 픽셀 맵을 지원하는 위젯에 다른 필드를 할당하려면 속성 보기를 사용해야 합니다.

## <a name="using-templates"></a>템플릿 사용

Studio에서 디자인하는 모든 화면 또는 자식 위젯 컬렉션을 새 화면 및 새 자식 컨트롤의 템플릿으로 사용할 수 있습니다. 일반적인 사용 사례인 창 유형 위젯 또는 기타 위젯 유형을 기반으로 템플릿을 만들 수 있습니다. 템플릿을 사용하는 것은 위젯을 복사하여 붙여넣는 것과 비슷합니다. 단, 기반이 되는 템플릿이 수정될 때 템플릿에서 파생된 모든 항목이 자동으로 수정됩니다. 파생된 화면 또는 템플릿의 상속된 인스턴스를 사용하여 작업하는 경우 템플릿 위젯 속성을 수정할 수 없습니다. 그러나 어떤 방식으로든 템플릿 속성을 수정하면 해당 템플릿을 참조하는 모든 인스턴스는 해당 템플릿에서 파생되므로 자동으로 업데이트됩니다.

템플릿을 반복되는 항목에 사용하는 경우의 또 다른 이점은 Studio에서 생성한 사양 파일의 크기가 일반적으로 반복되는 항목을 사용할 때마다 이러한 항목을 다시 만든 경우보다 더 작다는 것입니다.

화면 또는 자식 위젯 컬렉션을 템플릿으로 사용하도록 지정하려면 위젯 속성 보기에서 "템플릿" 확인란을 설정합니다. 템플릿 확인란을 설정하면 템플릿 위젯이 ***삽입|템플릿*** 풀다운 메뉴에 표시됩니다.

템플릿을 사용하는 예로, 단추 모음으로 사용되는 창을 정의할 수 있습니다. 이 창에는 여러 개의 자식 단추가 있을 수 있으며, 이 단추 모음은 다양한 화면에서 자주 사용됩니다. Studio 프로젝트 내에서 필요한 자식 단추를 포함하는 작은 독립 실행형 창을 정의하고, "button_bar"라는 이름을 이 창에 지정할 수 있습니다. 그런 다음, 이 창을 선택하고 "템플릿" 속성을 설정합니다. 다음으로, 이 단추 모음을 추가하려는 화면을 선택합니다. 삽입|템플릿|button_bar 메뉴 명령을 사용하여 button_bar 창의 인스턴스를 화면에 삽입합니다. 단추 모음의 위치는 변경할 수 있지만 대부분의 속성을 변경할 수는 없습니다. 그러나 미리 정의된 다른 GUIX 위젯 형식과 마찬가지로 button_bar 위젯(및 모든 자식 항목)을 사용할 수 있습니다. button_bar를 수정하려면 button_bar 템플릿을 선택하여 변경해야 합니다.

템플릿을 일반적으로 사용하는 또 다른 예로, 비슷한 화면이 많이 포함된 애플리케이션이 있습니다. 예를 들어 애플리케이션에는 공통 제목 표시줄, 채우기 색, 크기 등을 모두 공유하는 10개의 다른 화면이 있을 수 있습니다. 이 경우 제목 표시줄 자식 위젯을 포함하고 화면 크기, 채우기 색 및 기타 속성을 구성하는 템플릿 화면을 정의할 수 있습니다. 이 템플릿 화면이 정의되면 이 템플릿에서 10개의 다른 화면을 파생할 수 있습니다. 삽입|템플릿|\<base_screen> 메뉴 명령을 사용하면 템플릿 화면의 모든 자식 위젯 및 설정으로 화면이 시작됩니다. 템플릿 화면에서 파생된 각 화면은 템플릿의 복사본이 아니라 템플릿 화면의 파생된 인스턴스입니다. 그런 다음, 필요한 추가 콘텐츠를 보유하도록 각 파생된 화면을 사용자 지정할 수 있습니다.

생성된 사양 파일의 크기를 저장하는 것 외에도 템플릿을 사용하면 애플리케이션 모양에 대한 변경 내용을 더 쉽게 관리할 수 있습니다. 위의 예에서 10개의 비슷한 화면에 대한 배경색을 변경해야 한다고 가정합니다. 각 화면을 선택하고 채우기 색 설정을 변경하는 대신, 기본 템플릿을 선택하고 채우기 색만 변경하기만 하면 됩니다. 이 변경 내용은 파생된 모든 화면에 즉시 반영됩니다.

템플릿에 대한 추가 설명: 이벤트 처리 흐름이 유지되도록 해야 합니다. 즉, 기본 화면(공통 위젯 이벤트 처리용) 및 파생된 화면 모두에 대해 이벤트 처리기를 제공하는 경우 파생된 화면 이벤트 처리기는 기본 사례의 base_screen 이벤트 처리기를 호출해야 합니다. 이렇게 하면 기본 화면 이벤트 처리기가 이 템플릿 기반에서 파생된 모든 화면에 공통되는 위젯에서 생성한 이벤트를 처리할 수 있습니다.

## <a name="record-and-playback-macro"></a>매크로 기록 및 재생

매크로 기록 및 재생 함수는 키 입력과 마우스 이벤트를 기록하고 재생하는 데 도움이 됩니다.

매크로 파일에 기록하려면 ***매크로 기록** _ 도구 모음 단추를 선택하거나 _*_편집, 매크로 기록_*_ 을 선택할 수 있는 메뉴를 선택합니다. GUIX Studio는 매크로 파일의 경로 이름을 지정할 수 있는 _*_매크로 기록_*_ 대화 상자를 제공합니다. 이 항목이 선택되면 _*_기록_*_ 단추를 클릭하여 기록을 시작합니다. 기록이 완료되면 _*_매크로 기록_*_ 도구 모음 단추를 다시 선택하거나 _ *_편집, 매크로 종료_**를 선택할 수 있는 풀다운 메뉴를 사용하여 매크로 기록을 종료합니다.

매크로 파일을 재생하려면 ***매크로 재생** _ 도구 모음 단추를 선택하거나 _*_편집, 매크로 재생_*_ 명령을 선택할 수 있는 기본 풀다운 메뉴를 사용합니다. GUIX Studio는 실행할 이전에 기록된 매크로 파일을 지정할 수 있는 _ *_매크로 재생_** 대화 상자를 제공합니다.

글꼴 또는 이미지 추가와 같이 입력 또는 출력 파일을 선택하는 매크로를 기록하는 경우 마우스를 사용하여 파일 브라우저에서 선택하는 대신 키보드를 사용하여 파일 이름을 입력해야 합니다. 매크로 레코더는 마우스 및 키보드 이벤트를 기록하고 파일 브라우저는 시간이 지남에 따라 변경될 수 있으므로 파일을 그래픽으로 선택하는 것보다 파일 이름을 입력하는 것이 더 안정적입니다.

확대 함수는 대상 화면을 확대하여 표시하는 데 도움이 됩니다.

***구성|대상 보기|확대/축소** _ 메뉴 옵션에서 원하는 백분율 확대/축소 설정을 선택할 수 있습니다. _ *_도구 모음_**에도 확대/축소 단추가 있습니다.

## <a name="gridsnap-settings"></a>눈금/맞춤 설정

***눈금 및 맞춤 설정** _ 대화 상자에는 눈금 및 맞춤에 대한 일부 설정과 옵션이 포함되어 있습니다. _*_그림 29_*_ 에서는 _ *_구성|대상 보기|눈금/맞춤_** 메뉴를 선택한 경우의 _*_눈금 및 맞춤 설정_*_ 대화 상자를 보여 줍니다.

![눈금 및 맞춤 설정의 스크린샷](./media/guix-studio/image63.jpg)

**그림 29**

***눈금 표시** _ 옵션을 설정하면 대상 화면에 눈금이 표시됩니다. _*_눈금 간격_*_ 필드에서 눈금 증가값(픽셀)을 지정할 수 있습니다. _ *_눈금에 맞춤_** 옵션을 사용하면 위젯의 적절한 위치를 가져올 수 있습니다. 이 옵션을 설정하면 활성 맞춤이 활성화됩니다.

***눈금 및 맞춤*** 옵션이 사용하도록 설정되면 다음과 같습니다.

- 대상 보기에서 마우스로 개체를 끌면 개체가 눈금 증가값 단위로 이동합니다.
- 크기를 조정할 개체의 가장자리를 끌면 끌고 있는 가장자리가 눈금 위치에 맞춰집니다.
- 개체를 선택하고 위쪽/왼쪽/아래쪽/오른쪽 키를 사용하면 선택한 위젯이 눈금 증가값 단위로 이동합니다. 맞춤 증가값(픽셀)은 ***맞춤 간격*** 필드에서 지정할 수 있습니다.
