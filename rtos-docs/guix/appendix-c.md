# <a name="appendix-c---guix-widget-styles"></a>부록 C - GUIX 위젯 스타일

__***일반 스타일(대부분의 위젯 유형에 사용됨):***__

**GX_STYLE_BORDER_NONE**
  - 값: 0x00000000
  - 설명: 테두리 없이 위젯을 그리려면 이 스타일을 사용합니다.

**GX_STYLE_BORDER_RAISED**
  - 값: 0x00000001
  - 설명: 볼록 테두리를 사용하여 위젯을 그립니다.

**GX_STYLE_BORDER_RECESSED**
  - 값: 0x00000002
  - 설명: 오목 테두리를 사용하여 위젯을 그립니다.

**GX_STYLE_BORDER_THIN**
  - 값: 0x00000004
  - 설명: 1픽셀 너비의 테두리를 그립니다.

**GX_STYLE_BORDER_THICK** 
  - 값: 0x00000008
  - 설명: 두꺼운 테두리를 사용하여 위젯을 그립니다.

**GX_STYLE_BORDER_MASK**
  - 값: 0x0000000f
  - 설명: 위젯 스타일 멤버의 스타일 필드만 테스트하기 위해 사용되는 마스크 값입니다.

**GX_STYLE_TRANSPARENT**
  - 값: 0x10000000
  - 설명: 최소한 부분적으로 투명한 위젯을 만듭니다. 이 스타일은 반투명 pixelmap을 위젯 배경으로 그리는 위젯을 포함하여 위젯이 자신을 완전 불투명으로 그리지 않을 때 사용됩니다. 이 스타일 플래그는 위젯 배경 영역을 새로 고치기 위해 위젯 부모를 그려야 함을 GUIX에 알려줍니다.

**GX_STYLE_DRAW_SELECTED**
  - 값: 0x20000000
  - 설명: 선택한 상태 색상 및 글꼴을 사용하여 위젯을 그리도록 지정합니다. 다른 위젯 유형은 여러 방법으로 DRAW_SELECTED 스타일을 사용하여 위젯이 현재 선택되었음을 나타냅니다.

**GX_STYLE_ENABLED**
  - 값: 0x40000000
  - 설명: 위젯에서 사용자 입력 이벤트를 수락하고 출력 신호를 생성할 수 있도록 위젯을 사용으로 표시합니다.
  
**GX_STYLE_DYNAMICALLY_ALLOCATED**
  - 값: 0x80000000
  - 설명: 위젯을 만들 때 gx_system_memory_allocator 서비스를 사용하여 위젯 제어 블록 메모리가 동적으로 할당되고 위젯이 삭제되면 제어 블록 메모리가 해제됨을 나타냅니다.

**GX_STYLE_USE_LOCAL_ALPHA**
  - 값: 0x01000000
  - 설명: 위젯을 그릴 때 로컬 위젯 알파 값을 사용하도록 GUIX 그리기 함수에 지시합니다. 이 플래그는 일반적으로 내부 GUIX 논리에서 위젯 페이딩 애니메이션을 구현하는 데 사용됩니다.


__***텍스트 맞춤 스타일(텍스트를 그리는 모든 위젯에 적용되는 스타일):***__

**GX_STYLE_TEXT_LEFT**
  - 값: 0x00001000
  - 설명: 위젯 클라이언트 영역 내에서 왼쪽 맞춤으로 텍스트가 그려집니다.

**GX_STYLE_TEXT_RIGHT** 
  - 값: 0x00002000
  - 설명: 위젯 클라이언트 영역 내에서 오른쪽 맞춤으로 텍스트가 그려집니다.

**GX_STYLE_TEXT_CENTER**
  - 값: 0x00004000
  - 설명: 위젯 클라이언트 영역 내에서 가운데 맞춤으로 텍스트가 그려집니다.

**GX_STYLE_TEXT_COPY**
  - 값: 0x00008000
  - 설명: 기본적으로 텍스트를 그리는 위젯은 애플리케이션에서 전달된 텍스트에만 포인터를 유지합니다. 문자열 테이블 내에 정의되는 정적으로 정의된 텍스트의 경우 위젯이 할당된 텍스트의 프라이빗 복사본을 만들 이유가 없습니다. 하지만 위젯에 할당된 텍스트가 sprint() 또는 gx_utility_ltoa와 같은 함수를 사용하여 동적으로 생성된 경우 할당된 텍스트의 고유한 프라이빗 복사본을 유지하도록 위젯에 지정하는 것이 편리한 경우가 많습니다. 이렇게 하면 애플리케이션이 텍스트 문자열을 정의할 때, 즉 동적으로 정의된 텍스트를 사용하는 각 텍스트 위젯에 대해 애플리케이션이 정적으로 정의된 문자 배열을 정의해야 하는 경우, 자동 또는 임시 변수를 사용할 수 있습니다. 이 스타일 플래그가 설정되면 위젯은 gx_system_memory_allocator 함수를 사용하여 할당된 문자열의 프라이빗 복사본을 저장하는 데 필요한 메모리 블록을 동적으로 할당합니다. 따라서 이 스타일 플래그 사용은 memory_deallocator 함수를 정의하는 애플리케이션을 기반으로 합니다. 설정된 뒤에는 GX_STYLE_TEXT_COPY를 지우지 않아야 합니다. 그렇지 않으면 예측할 수 없는 결과가 발생합니다.

__***단추 스타일(GUIX 단추 위젯 유형에만 적용):***__

**GX_STYLE_BUTTON_PUSHED**
  - 값 0x00000010
  - 설명: 단추를 눌렀거나 선택한 상태임을 나타냅니다.

**GX_STYLE_BUTTON_TOGGLE**
  - 값 0x00000020
  - 설명: 클릭 이벤트가 발생할 때마다 단추 상태가 푸시된 상태와 푸시 취소된 상태로 전환됩니다. 이 스타일은 일반적으로 “확인란” 스타일 단추와 함께 사용됩니다.

**GX_STYLE_BUTTON_RADIO**
  - 값 0x00000040
  - 설명: 이 스타일은 단추가 배타적임을 나타내며 이 단추를 선택하면 모든 형제 단추의 선택을 취소합니다. 이 스타일은 일반적으로 “라디오 단추” 스타일 단추와 함께 사용됩니다.

**GX_STYLE_BUTTON_EVENT_ON_PUSH**
  - 값: 0x00000080
  - 설명: 단추가 처음 푸시될 때 클릭 이벤트가 발생함을 나타냅니다. 기본 작업은 단추를 해제할 때 클릭 이벤트를 생성하는 것입니다.

**GX_STYLE_BUTTON_REPEAT**
  - 값 0x00000100
  - 설명: 단추가 푸시된 상태로 유지될 때 단추가 단추 부모에게 반복되는 클릭 이벤트를 전송해야 함을 나타냅니다.

__***목록 스타일(GUIX 목록 위젯 유형에만 적용):***__

**GX_STYLE_CENTER_SELECTED** 
  - 값: 0x00000010
  - 설명: 예약됨

**GX_STYLE_WRAP**
  - 값 0x00000020
  - 설명: 목록이 시작 또는 끝 목록 인덱스를 지나서 드래그되거나 스크롤될 때 목록 자식은 처음부터 끝까지 래핑됩니다.

**GX_STYLE_FLICKABLE**
  - 값: 0x00000040
  - 설명: 예약됨

__***Pixelmap 단추 및 아이콘 단추 스타일:***__

**GX_STYLE_HALIGN_CENTER**
  - 값: 0x00010000
  - 설명: 가로 축에서 단추 경계 내에 단추 pixelmap이 가운데 맞춤으로 정렬되어야 합니다.

**GX_STYLE_HALIGN_LEFT**
  - 값: 0x00020000
  - 설명: 가로 축에서 단추 경계 내에 단추 pixelmap이 왼쪽 맞춤으로 정렬되어야 합니다.

**GX_STYLE_HALIGN_RIGHT**
  - 값 0x00040000
  - 설명: 가로 축에서 단추 경계 내에 단추 pixelmap이 오른쪽 맞춤으로 정렬되어야 합니다.

**GX_STYLE_VALIGN_CENTER**
  - 값 0x00080000
  - 설명: 세로 축에서 단추 경계 내에 단추 pixelmap이 가운데 맞춤으로 정렬되어야 합니다.

**GX_STYLE_VALIGN_TOP**
  - 값: 0x00100000
  - 설명: 세로 축에서 단추 경계 내에 단추 pixelmap이 위쪽 맞춤으로 정렬되어야 합니다.

**GX_STYLE_VALIGN_BOTTOM**
  - 값: 0x00200000
  - 설명: 가로 세로 축에서 단추 경계 내에 단추 pixelmap이 아래쪽 맞춤으로 정렬되어야 합니다.

__***슬라이더 스타일(GX_SLIDER 및 파생된 위젯 유형에만 적용):***__

**GX_STYLE_SHOW_NEEDLE**
  - 값: 0x00000200
  - 설명: 슬라이더가 니들 표시기를 그리려면 이 스타일이 포함되어야 합니다. 애플리케이션에서 슬라이더 니들을 사용 중지하거나 사용자 지정 니들 표시기를 그리려면 이 스타일을 사용 중지할 수 있습니다.

**GX_STYLE_SHOW_TICKMARKS**
  - 값: 0x00000400
  - 설명: 이 스타일을 사용하는 경우 슬라이더 위젯이 파선 눈금 표시선에 대한 소프트웨어 그리기를 수행합니다.

**GX_STYLE_SLIDER_VERTICAL**
  - 값 0x00000800
  - 설명: 세로 슬라이더를 만들려면 이 스타일 플래그를 설정하고, 가로 슬라이더를 만들려면 이 스타일 플래그를 지웁니다.

__***스프라이트 스타일(GX_SPRITE 위젯 유형에만 적용):***__

**GX_STYLE_SPRITE_AUTO**
  - 값: 0x00000010
  - 설명: 스프라이트 위젯에 GX_EVENT_SHOW 이벤트가 수신되었을 때 스프라이트 애니메이션이 자동으로 실행됩니다.

**GX_STYLE_SPRITE_LOOP**
  - 값: 0x00000020
  - 설명: 이 스타일을 사용하면 애플리케이션에서 스프라이트를 중지할 때까지 스프라이트 위젯에 스프라이트 애니메이션 프레임이 연속으로 재생됩니다.

__***Pixelmap 슬라이더 스타일:***__

**GX_STYLE_TILE_BACKGROUND**
  - 값 0x00001000
  - 설명: 슬라이더 배경 이미지가 바둑판식으로 배열되어 스프라이트 경계 상자를 채웁니다. 이렇게 하면 작은 세로 또는 가로 스트라이프 이미지를 사용하여 슬라이더 배경을 채울 수 있습니다.

__***추가 진행률 표시줄 스타일:***__

**GX_STYLE_PROGRESS_PERCENT**
  - 값: 0x00000010
  - 설명: 이 스타일이 설정되면 진행률 표시줄에서 막대 값이 원시 값 대신 백분율로 그려집니다. 텍스트는 진행률 표시줄 경계 상자의 가운데에 표시됩니다.

**GX_STYLE_PROGRESS_TEXT_DRAW**
  - 값: 0x00000020
  - 설명: 진행률 표시줄에서 가운데 표시된 십진수 텍스트로 현재 진행률 표시줄 값을 그립니다.

**GX_STYLE_PROGRESS_VERTICAL**
  - 값: 0x0000040
  - 설명: 진행률이 세로 방향으로 표시됩니다. 기본값은 가로 방향입니다.

**GX_STYLE_PROGRESS_SEGMENT_FILL**:
  - **값**: 0x00000100
  - 설명: 진행률 표시줄 값이 단색 채우기 대신 세그먼트로 채워진 사각형으로 표시됩니다.

__***추가 방사형 진행률 표시줄 스타일:***__

**GX_STYLE_RADIAL_PROGRESS_ALIAS**
  - 값: 0x00000200
  - 설명: 앤티앨리어싱된 브러시 스타일을 사용하여 방사형 진행률 표시줄을 그립니다. 이렇게 하면 CPU 대역폭이 더 많이 필요하지만 모양이 더 좋아집니다. 성능이 낮은 CPU 대상의 경우 이 스타일 플래그를 지우면 그리기 속도가 빨라집니다.

**GX_STYLE_RADIAL_PROGRESS_ROUND**
  - 값: 0x00000400
  - 설명: 방사형 진행률 표시줄 호를 그릴 때 둥근 선 끝 브러시 스타일을 사용합니다. 기본값은 사각형 선 끝입니다.

__***추가 텍스트 입력 스타일:***__

**GX_STYLE_ CURSOR_BLINK**
  - 값: 0x00000040
  - 설명: 텍스트 입력 위젯 커서가 안정되지 않고 깜박입니다.

**GX_STYLE_ CURSOR_ALWAYS**
  - 값: 0x00000080
  - 설명 텍스트 입력 위젯 커서는 일반적으로 위젯에 입력 포커스가 있을 때만 표시됩니다. 이 스타일 플래그는 입력 포커스에 관계없이 커서를 항상 표시합니다.

**GX_STYLE_TEXT_INPUT_NOTIFY_ALL**
  - 값: 0x00000100
  - 설명: 이 스타일 플래그는 텍스트 입력 위젯에 키 누름 이벤트가 수신될 때마다 GX_EVENT_TEXT_EDITED 이벤트를 설정합니다.

__***텍스트 스크롤 휠 스타일:***__
