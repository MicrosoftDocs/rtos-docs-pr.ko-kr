# <a name="chapter-8-notes-on-editing-specific-widget-types"></a>챕터 8: 특정 위젯 형식 편집에 관한 참고 사항

GUIX Studio를 사용하면 사용자가 애플리케이션 UI 화면을 구성하는 GUIX 위젯을 쉽게 만들고 수정할 수 있습니다. 이러한 편집 방법은 대부분 직관적이고 명확합니다. 예를 들어 위젯의 크기를 조정하려면 마우스로 위젯을 선택하고 위젯 테두리를 끌어서 놓을 수 있습니다. 선택한 위젯의 왼쪽/위쪽/너비/높이 속성 필드에 직접 입력할 수도 있습니다.

이러한 위젯은 해당 기능 지원에서 약간 더 정교하기 때문에 특정 위젯 형식에는 고급 편집 프로세스가 필요합니다.

이 챕터는 보다 복잡한 위젯 형식의 고급 편집 기능에 대한 참고 사항을 제공합니다. 이러한 참고 사항은 GUIX Studio의 기능이 GUIX 라이브러리 내에서 제공되는 위젯 형식과 함께 진행될 때 확장됩니다.

## <a name="rich-text-view"></a>서식 있는 텍스트 보기

이 위젯은 인라인 텍스트 서식 지정 코드를 지원하는 서식 있는 텍스트를 표시하는 데 사용됩니다. 이러한 서식 지정 코드에는 굵은 글꼴, 기울임꼴 및 기타 여러 항목이 포함됩니다. 시작하려면 *프로젝트 보기* 또는 *대상 보기* 에서 선택한 부모를 마우스 오른쪽 단추로 클릭하고 메뉴 **삽입/텍스트/서식 있는 텍스트 보기** 를 선택하여 서식 있는 텍스트 보기 위젯을 삽입합니다.

모든 위젯 형식에서 지원하는 표준 속성 집합 외에도 서식 있는 텍스트 보기 위젯 형식은 추가 속성을 지원합니다.

### <a name="additional-properties"></a>추가 속성

| 속성 | 의미 |
|----------|---------|
| 텍스트 맞춤 | 기본 텍스트 맞춤 |
| 일반 글꼴| 기본 텍스트 글꼴 |
| 굵은 글꼴  | 굵은 글꼴로 태그가 지정된 텍스트의 텍스트 글꼴 |
| 기울임꼴 글꼴| 기울임꼴로 태그가 지정된 텍스트의 텍스트 글꼴|
| 굵은 기울임꼴 글꼴| 굵은 글꼴 및 기울임꼴로 태그가 지정된 텍스트의 텍스트 글꼴 |
| 개인 텍스트 복사 | 위젯이 할당된 텍스트의 개인적인 복사본을 유지해야 하는지 여부를 확인해야 함 |
| 공백 | 위젯과 표시되는 텍스트 사이의 여백 너비(픽셀)입니다. |
| 줄 공간 | 텍스트의 두 줄 사이에 있는 공간(픽셀)입니다. |
|||

### <a name="the-rich-text-formatting-codes"></a>서식 있는 텍스트 서식 지정 코드

텍스트 서식 지정에 대해 지원되는 서식 코드는 다음과 같습니다.

|태그|의미|
|---|---|
|\<b>\</b> | 사용자 지정 굵은 글꼴 ID로 텍스트 글꼴 렌더링|
|\<i>\</i> | 사용자 지정 기울임꼴 ID로 텍스트 글꼴 렌더링|
|\<u>\</u> | 밑줄이 그어진 텍스트 렌더링|
|\<f GX_FONT_ID>\</f> | 지정된 글꼴 ID를 사용하여 텍스트 렌더링 |
|\<c GX_COLOR_ID>\</c> | 지정된 색 ID를 사용하여 텍스트 렌더링|
|\<hc GX_COLOR_ID>\</hc> | 지정된 배경색 ID를 사용하여 텍스트 렌더링|
|\<align left/right/center>\</align> | 텍스트 맞춤 할당|
|||

GUIX Studio 내에서 서식 있는 텍스트 문자열을 편집하는 방법에는 두 가지가 있습니다.

- 권장되며 가장 간단한 편집 방법인 서식 있는 텍스트 편집 대화 상자를 사용합니다.
- 문자열 테이블 편집기 대화 상자를 사용하여 앞의 표에 나와 있는 서식 태그를 수동으로 삽입할 수 있습니다.

*대상 보기* 에서 서식 있는 텍스트 보기 위젯을 선택한 후 *속성 보기* 에서 **서식 있는 텍스트 편집** 단추를 선택하여 그림 8.1에 표시된 서식 있는 텍스트 편집 대화 상자를 호출합니다.

![GUIX Studio 서식 있는 텍스트 편집 대화 상자의 스크린샷입니다.](./media/guix-studio/edit_rich_text_dialog.png)

**그림 8.1**

왼쪽 창은 서식 있는 텍스트 편집 필드입니다. 도구 모음 아이콘을 사용하여 필요한 태그를 삽입할 수 있습니다. 편집 필드에서 텍스트 블록을 선택한 다음 도구 모음 단추를 선택하여 선택한 텍스트 블록에 필요한 스타일과 색을 적용합니다. 서식 있는 텍스트 편집 대화 상자는 테스트 문자열에 서식 코드를 삽입하는 쉬운 방법입니다. 필요한 경우 이러한 태그를 수동으로 삽입하거나 런타임 시 생성할 수도 있습니다.

오른쪽 창은 대상 보기에서 텍스트가 렌더링되는 방식을 보여주는 위젯 미리 보기입니다. 위젯 미리 보기의 배경색이 고정되어 있으며 이 색은 대상 보기에서 위젯의 할당된 배경색과 일치하지 않을 수 있습니다.

리소스 ID 이름은 특정 글꼴 또는 색 리소스를 참조하는 서식 있는 텍스트에 사용됩니다. 서식 있는 텍스트 문자열에서 참조한 후 글꼴 또는 색의 리소스 이름이 변경된 경우, GUIX Studio는 리소스 이름 변경 내용을 반영하도록 서식 있는 텍스트 문자열을 자동으로 업데이트합니다. 반면 서식 있는 텍스트 위젯에 의해 참조되는 글꼴 또는 색 리소스를 삭제하는 경우, 삭제된 리소스 ID 이름을 제거하거나 변경하려면 영향을 받는 서식 있는 텍스트를 수동으로 편집해야 합니다.

이 대화 상자에서 저장 단추를 선택하면 정의한 서식 있는 텍스트 문자열이 프로젝트 문자열 테이블에 추가됩니다.

## <a name="string-scroll-wheel"></a>문자열 스크롤 휠

문자열 스크롤 휠 위젯은 문자열 배열의 표시를 지원합니다. 이러한 문자열은 동적으로 할당될 수 있으며, 애플리케이션에서 여러 언어를 지원하는 경우에는 활성 문자열 테이블에서 할당된 문자열을 가져올 수 있습니다.

문자열 스크롤 휠 위젯은 문자열 배열을 지원합니다. 그림 8.2에 표시된 문자열 스크롤 휠 편집 대화 상자는 사용자가 이 문자열 배열을 할당할 수 있도록 제공됩니다.

![GUIX Studio 문자열 스크롤 휠 편집 대화 상자 스크린샷입니다.](./media/guix-studio/string_scroll_wheel_edit.png)

**그림 8.2**

이 대화 상자를 호출하려면 *대상 보기* 또는 *프로젝트 보기* 내에서 문자열 스크롤 휠 위젯을 선택합니다. 이 위젯 형식을 선택하면 *속성 보기* 에 **문자열 편집** 단추가 포함됩니다. 문자열 스크롤 휠 편집 대화 상자를 호출하려면 이 단추를 선택합니다.

드롭다운 목록에서 문자열 ID를 선택하거나 오른쪽의 텍스트 필드에 새 문자열 값을 입력하여 각 텍스트 인덱스에 문자열을 할당할 수 있습니다. 변경 작업을 마치면 새 문자열이나 수정된 문자열이 활성 문자열 테이블에 저장됩니다.

## <a name="sprite"></a>스프라이트

스프라이트 위젯은 애니메이션 효과를 제공하는 이미지 시퀀스를 표시하는 데 사용됩니다. 스프라이트 위젯에는 프레임 목록이 필요합니다. 이 목록은 프레임의 각 이미지에 적용되는 고유 매개 변수 및 이미지 ID의 배열입니다. 스프라이트 위젯의 이 프레임 목록을 빌드하기 위해 그림 8.3에 표시된 스프라이트 프레임 편집 대화 상자가 제공됩니다.

![GUIX Studio 스프라이트 프레임 편집 대화 상자의 스크린샷입니다.](./media/guix-studio/edit_sprite_frames.png)

**그림 8.3**

이 대화 상자를 호출하려면 *대상 보기* 또는 *프로젝트 보기* 내에서 스프라이트 위젯을 선택합니다. 이 위젯 형식을 선택하면 *속성 보기* 에 **프레임 목록 편집** 단추가 포함됩니다. 문자열 스크롤 휠 편집 대화 상자를 호출하려면 이 단추를 선택합니다.

*총 스프라이트 프레임 수* 필드는 스프라이트 위젯에 표시되는 정수 전체 프레임 수를 입력할 수 있는 입력 필드입니다. 이미지는 프레임 목록 내에서 재사용할 수 있으며, 모든 이미지가 고유해야 하는 것은 아닙니다.

스프라이트 프레임 ID 필드는 1에서 총 프레임 수 사이의 범위에 해당하는 프레임 인덱스 선택 값입니다. 이 값을 증가시키거나 감소시켜 하나의 스프라이트 프레임에서 다음으로 이동합니다.

각 스프라이트 프레임에는 여러 매개 변수가 있습니다. 첫 번째는 백그라운드 작업입니다. 이 필드는 인기 있는 GIF 애니메이션 형식의 기능을 모방합니다. 선택할 수 있는 항목은 다음과 같습니다.

- ‘작업 없음’은 현재 프레임 이미지를 이전 프레임 이미지 위에 그리는 것을 의미합니다.
- ‘첫 번째 pixelmap 복원’은 인덱스 1 pixelmap을 현재 pixelmap 앞에 그리는 것을 의미합니다.
- ‘단색 채우기’는 현재 프레임을 그리기 전에 스프라이트 배경을 스프라이트 배경색으로 채우는 것을 의미합니다.

pixelmap ID 필드를 사용하여 이전에 프로젝트 리소스에 추가한 모든 pixelmap을 선택할 수 있습니다. 여러 프레임에 동일한 pixelmap ID를 사용할 수 있습니다. 예를 들어 스프라이트 애니메이션은 다른 스프라이트 이미지를 사용하는 대신 또는 추가로 x 및 y 오프셋 필드를 사용하여 이미지의 움직임을 활용할 수 있습니다.

알파 값 필드는 전체 pixelmap 그리기에 적용됩니다. 이 필드는 8bpp 색 깊이 이상으로 실행하는 경우에만 효과가 있습니다. 알파 값 0은 완전히 투명하고 알파 값 255는 불투명합니다.

프레임 x 오프셋 및 프레임 y 오프셋 필드를 사용하여 현재 pixelmap이 그려지는 스프라이트 프레임 내에서 오프셋을 지정할 수 있습니다. 즉, 그려진 각 이미지가 스프라이트 위젯의 전체 크기일 필요는 없습니다.

지연 기간은 다음 스프라이트 프레임으로 이동하기 전에 지연할 시간을 지정합니다. 이 값은 틱 단위이며 기본 GUIX/ThreadX 타이머 구성에서 각 틱은 50ms를 나타냅니다.

스프라이트 프레임 편집 대화 상자에서 변경 내용을 저장하면 GUIX Studio에서 출력 사양 파일 생성의 일부로 전체 프레임 목록 배열을 생성할 수 있습니다.
